# El NHS británico cierra repositorios de código abierto ante la amenaza de LLM

**Fecha:** 2026-05-02
**Categoría:** ia
**Tags:** llms, codigo, linux
**Título original:** Eden: NHS goes to war against open source

---

## Introducción

El Servicio Nacional de Salud (NHS) del Reino Unido está a punto de cerrar la mayoría de sus repositorios de código abierto debido a la mejora de las herramientas de escaneo de seguridad. Esta decisión ha sido criticada por expertos en seguridad y desarrollo de software.

## ¿Qué es?

El NHS ha decidido cerrar casi todos sus repositorios de código abierto debido a la mejora de las herramientas de escaneo de seguridad de los modelos de lenguaje de lógica (LLM). Esto significa que el código fuente de la mayoría de sus proyectos se volverá privado.

## ¿Cómo funciona?

Los LLM, como Mythos de Anthropic, pueden encontrar vulnerabilidades de seguridad en el código fuente de manera más efectiva. Sin embargo, la mayoría de los repositorios de código abierto del NHS contienen datos, herramientas internas y documentación, que no son vulnerables a estas herramientas.

## ¿Por qué importa?

Esta decisión importa porque contradice la política de código abierto del NHS y puede afectar a la comunidad de desarrollo de software. Además, la experiencia del NHS con la aplicación de contacto de Covid-19 ha demostrado que el código abierto no es un problema de seguridad.

## Consejo técnico

Si eres desarrollador de software en el NHS, considera crear un plan de migración para mantener tus repositorios de código abierto. Puedes utilizar herramientas como GitLab o GitHub para mantener tus proyectos privados mientras sigues utilizando código abierto.
