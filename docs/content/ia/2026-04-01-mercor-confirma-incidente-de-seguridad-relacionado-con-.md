# Mercor confirma incidente de seguridad relacionado con LiteLLM

**Fecha:** 2026-04-01
**Categoría:** ia
**Tags:** llms, ia-local, linux
**Título original:** Mercor says it was hit by cyberattack tied to compromise of open-source LiteLLM project

---

## Introducción

La empresa de inteligencia artificial Mercor ha confirmado un incidente de seguridad relacionado con la vulnerabilidad de la biblioteca LiteLLM, una herramienta de código abierto utilizada para el desarrollo de modelos de lenguaje. La empresa asegura que se está investigando y que se están tomando medidas para contener y remediar el incidente.

## ¿Qué es?

LiteLLM es una biblioteca de código abierto que se utiliza para desarrollar modelos de lenguaje. Es una herramienta popular en la comunidad de inteligencia artificial y se utiliza para entrenar modelos de lenguaje que pueden entender y generar texto. La biblioteca se utiliza en proyectos de inteligencia artificial como Mercor, que utiliza LiteLLM para entrenar modelos de lenguaje que pueden entender y responder a preguntas de búsqueda de trabajo.

## ¿Cómo funciona?

LiteLLM se utiliza para entrenar modelos de lenguaje mediante el uso de técnicas de aprendizaje automático. La biblioteca se utiliza para cargar y procesar grandes cantidades de datos de texto, que luego se utilizan para entrenar un modelo de lenguaje que pueda entender y generar texto. La biblioteca también se utiliza para evaluar el rendimiento de los modelos de lenguaje y para identificar áreas de mejora.

## ¿Por qué importa?

El incidente de seguridad en Mercor es importante porque pone en riesgo la seguridad de la información de los usuarios de la plataforma. La vulnerabilidad de LiteLLM puede permitir a atacantes acceder a la información de los usuarios y utilizarla para fines maliciosos. La empresa está tomando medidas para contener y remediar el incidente y asegura que se está investigando para identificar la causa raíz del problema.

## Consejo técnico

Si estás utilizando LiteLLM en tu proyecto de inteligencia artificial, es importante revisar tu código y asegurarte de que estás utilizando la última versión de la biblioteca. También es importante seguir las mejores prácticas de seguridad y utilizar herramientas de seguridad para proteger tu código y tus datos.

```bash
pip install --upgrade lite-llm
```
