# Redox OS prohíbe contribuciones con LLMs: un paso hacia la transparencia

**Fecha:** 2026-04-09
**Categoría:** ia
**Tags:** ia-local, linux, codigo
**Título original:** Redox OS Establishes AI Policy To Forbid Contributions Made Using LLMs

## Introducción

El sistema operativo Redox OS ha establecido una política de no aceptar contribuciones generadas por modelos de lenguaje grande (LLMs). Esto busca garantizar la transparencia y la autenticidad de los cambios en el código. La medida se suma a una serie de mejoras en la documentación y la estabilidad del sistema.

## ¿Qué es?

Redox OS es un sistema operativo open-source basado en Rust, diseñado para ser seguro, estable y fácil de usar. Se enfoca en proporcionar una experiencia de usuario intuitiva y una interfaz de programación de aplicaciones (API) robusta.

## ¿Cómo funciona?

El sistema operativo utiliza un enfoque modular para organizar sus componentes, lo que permite una mayor flexibilidad y escalabilidad. La política de no aceptar contribuciones con LLMs se implementará a través de un análisis de código que detectará automáticamente las contribuciones generadas por modelos de lenguaje grande.

## ¿Por qué importa?

Esta medida es importante porque garantiza la transparencia y la autenticidad de los cambios en el código. Esto es especialmente relevante en un proyecto open-source, donde la confianza de los desarrolladores y los usuarios depende de la integridad del código.

## Consejo técnico

Si deseas contribuir al proyecto Redox OS, asegúrate de utilizar herramientas de desarrollo que no utilicen LLMs para generar código. Por ejemplo, puedes utilizar el comando `git diff` para comparar cambios en el código y detectar automáticamente contribuciones generadas por modelos de lenguaje grande.

```bash
git diff
```
