# SDL prohíbe contribuciones de código generadas por LLM y modelos de IA

**Fecha:** 2026-04-16
**Categoría:** ia
**Tags:** llms, ia-local, linux
**Título original:** SDL Adds Policy To Forbid LLM/AI Generated Code Contributions

---

## Introducción

La biblioteca SDL, utilizada en juegos cross-platform y parte del Runtime de Steam, ha establecido una política para bloquear contribuciones de código generadas por modelos de lenguaje grande (LLM) y inteligencia artificial (IA). Esto se debe a la preocupación de que el código generado por estas herramientas no sea compatible con la licencia Zlib o pueda introducir términos de licencia conflictivos.

## ¿Qué es?

La SDL (Simple DirectMedia Layer) es una biblioteca de código abierto utilizada en juegos cross-platform y parte del Runtime de Steam. Es una herramienta fundamental en la industria del juego y se utiliza para crear juegos que pueden ejecutarse en múltiples plataformas, incluyendo Windows, macOS y Linux.

## ¿Cómo funciona?

La SDL utiliza una arquitectura modular que permite a los desarrolladores crear juegos que pueden interactuar con hardware de diferentes plataformas. La biblioteca proporciona una capa de abstracción entre el juego y el hardware, lo que permite a los desarrolladores escribir código que sea independiente de la plataforma.

## ¿Por qué importa?

La decisión de la SDL de prohibir contribuciones de código generadas por LLM y modelos de IA es importante porque puede afectar la calidad y la compatibilidad del código. El uso de herramientas de IA para generar código puede llevar a la introducción de errores y a problemas de compatibilidad que pueden ser difíciles de detectar y corregir.

## Consejo técnico

Si deseas contribuir al proyecto SDL, asegúrate de que tus cambios sean manuales y no generados por herramientas de IA. Puedes utilizar herramientas como GitHub Copilot con precaución, pero asegúrate de revisar y validar manualmente el código generado.

```bash
Para revisar la política de contribuciones de la SDL, puedes acceder a la página de GitHub de la biblioteca y buscar la sección de 'AGENTES.md'.
```
