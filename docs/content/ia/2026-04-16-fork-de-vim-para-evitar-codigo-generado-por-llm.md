## Introducción

La proliferación de Modelos de Lenguaje de Gran Escala (LLM) ha generado rechazo en algunas personas. Dos nuevos forks de Vim buscan ofrecer un entorno de edición sin código generado por LLM. Estos proyectos buscan proporcionar alternativas estables, siempre y cuando haya suficiente interés.

## ¿Qué es?

Vim es un editor de texto popular que ha sido modificado para crear dos nuevos forks: EVi y Vim Classic. EVi busca ser una versión moderna de Vim sin contribuciones asistidas por LLM, mientras que Vim Classic busca proporcionar una versión de larga duración de Vim 8. Estos proyectos buscan ofrecer un entorno de edición sin código generado por LLM.

## ¿Cómo funciona?

Los forks de Vim se basan en la versión 8 del editor y buscan eliminar cualquier contribución asistida por LLM. ## ¿Por qué importa?

Estos proyectos importan porque ofrecen alternativas estables a los usuarios que buscan evitar el código generado por LLM. También buscan proporcionar un entorno de edición sin contribuciones asistidas por LLM, lo que puede ser beneficioso para los desarrolladores que buscan mantener su código libre de influencias externas.

## Consejo técnico

Si estás interesado en utilizar EVi o Vim Classic, puedes descargar el código fuente de estos proyectos y compilarlos en tu sistema. También puedes contribuir a estos proyectos proporcionando código y mejorando la documentación.

```bash
git clone https://github.com/evim/evim.git
```
