## Introducción

La próxima versión de ROS 2, Lyrical Luth, traerá mejoras en comunicaciones y mantendrá a Fast DDS como middleware por defecto. Esta elección se debe a la confianza de la comunidad en la escalabilidad, fiabilidad y rendimiento de Fast DDS.

## ¿Qué es?

ROS 2 es una plataforma de software de código abierto para el desarrollo de sistemas de robótica y automatización. Fast DDS es un middleware de comunicaciones que proporciona una capa de abstracción para la comunicación entre nodos en la red. Lyrical Luth es la próxima versión de ROS 2.

## ¿Cómo funciona?

Fast DDS se utiliza como middleware por defecto en ROS 2, lo que significa que proporciona una capa de abstracción para la comunicación entre nodos en la red. Lyrical Luth traerá mejoras en la comunicación, incluyendo soporte para buffers nativos y transportes acelerados en memoria y GPU.

## ¿Por qué importa?

La elección de Fast DDS como middleware por defecto se debe a su escalabilidad, fiabilidad y rendimiento. Esto es especialmente importante en aplicaciones de robótica y automatización, donde la comunicación entre nodos es crítica. Lyrical Luth también traerá mejoras en la comunicación, lo que permitirá a los desarrolladores crear sistemas más eficientes y escalables.

## Consejo técnico

Si estás desarrollando un sistema de robótica o automatización con ROS 2, considera utilizar Fast DDS como middleware para aprovechar sus beneficios de escalabilidad, fiabilidad y rendimiento. Puedes configurarlo utilizando el comando `ros2 run eprosima_fastdds_demo demo`.

```bash
ros2 run eprosima_fastdds_demo demo
```
