# Honda Civics y la vulnerabilidad de actualización de software

**Fecha:** 2026-06-16
**Categoría:** linux
**Tags:** robotica, linux, codigo
**Título original:** Honda Civics and Installing Software With Android Test Keys

---

## Introducción

Un ingeniero ha descubierto una vulnerabilidad en los sistemas de entretenimiento de algunos modelos de Honda Civic, que permiten actualizar el software mediante un USB. Esto podría ser una solución para propietarios de vehículos con sistemas de entretenimiento obsoletos.

## ¿Qué es?

La vulnerabilidad se conoce como EvilValet y permite a cualquier persona con acceso físico al puerto USB del vehículo actualizar el software del sistema de entretenimiento. Esto se logra utilizando claves de prueba de Android Open Source Project (AOSP) que se encuentran en el sistema de archivos.

## ¿Cómo funciona?

El proceso de actualización se realiza mediante la inserción de un USB con el software actualizado y el uso de las claves de prueba de AOSP. El sistema de entretenimiento del vehículo se puede actualizar a una versión más reciente, lo que podría mejorar su seguridad y funcionalidad.

## ¿Por qué importa?

Esta vulnerabilidad importa porque permite a los propietarios de vehículos con sistemas de entretenimiento obsoletos actualizar su software de manera segura. Esto podría ser especialmente útil para aquellos que tienen problemas con su sistema de entretenimiento o que desean mejorar su seguridad.

## Consejo técnico

Si tienes un vehículo con un sistema de entretenimiento Android, verifica si se aplica la vulnerabilidad EvilValet. Puedes encontrar más información en el proyecto de GitHub de Eric McDonald, el ingeniero que descubrió la vulnerabilidad.
