# Microchip lanza el SAM9X75, un MCU ARM9 para aplicaciones automotrices

**Fecha:** 2026-03-27
**Categoría:** linux
**Tags:** robotica, embebidos, linux
**Título original:** Microchip SAM9X75 Hybrid automotive MCU – Surprisingly ARM9 is still a thing in 2026

## Introducción

Microchip ha lanzado el SAM9X75, un microcontrolador de código abierto basado en el núcleo ARM9, diseñado específicamente para aplicaciones automotrices y HMI (Interfaz de Usuario Humano) en vehículos eléctricos y convencionales.

## ¿Qué es?

El SAM9X75 es un microcontrolador System-in-Package (SiP) que combina un núcleo ARM926EJ-S con una memoria DDR2 o DDR3L integrada. Ofrece una velocidad de reloj de hasta 800 MHz y una memoria de 512 MB, 1 GB o 2 GB, dependiendo de la variante.

## ¿Cómo funciona?

El SAM9X75 cuenta con una arquitectura de SoC (Sistema en un Chip) que incluye un núcleo ARM9, una memoria DDR2 o DDR3L, un controlador de imagen, un controlador de audio y una interfaz de red Gigabit Ethernet. También dispone de varios puertos de expansión, como SPI, I2C, UART y CAN-FD.

## ¿Por qué importa?

El SAM9X75 es importante porque ofrece una solución de código abierto para aplicaciones automotrices y HMI, lo que permite a los desarrolladores crear sistemas de infotainment y sistemas de control de vehículos eléctricos y convencionales de manera más eficiente y económica.

## Consejo técnico

Si estás desarrollando un sistema de infotainment para un vehículo eléctrico, considera utilizar el SAM9X75 como plataforma de desarrollo, ya que ofrece una gran cantidad de recursos y puertos de expansión para integrar diferentes componentes y sensores.
