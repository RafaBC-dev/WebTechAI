# Google detalla nuevo proceso para instalar aplicaciones no verificadas en Android

**Fecha:** 2026-03-20
**Categoría:** linux
**Tags:** android, seguridad, desarrollo
**Título original:** Google details new 24-hour process to sideload unverified Android apps (Ars

## Introducción

Google ha anunciado un nuevo proceso para instalar aplicaciones no verificadas en Android, que requiere una serie de pasos y una espera de 24 horas. Este cambio busca mejorar la seguridad del sistema operativo.

## ¿Qué es?

El proceso de instalación de aplicaciones no verificadas en Android se ha modificado para requerir una serie de pasos y una espera de 24 horas. Esto busca mejorar la seguridad del sistema operativo y proteger a los usuarios de aplicaciones maliciosas.

## ¿Cómo funciona?

Para instalar aplicaciones no verificadas, los usuarios deben habilitar las opciones de desarrollador, permitir paquetes no verificados y confirmar que entienden los riesgos. Luego, deben esperar 24 horas y luego seleccionar la opción de instalación temporal o indefinida.

## ¿Por qué importa?

Este cambio es importante porque protege a los usuarios de aplicaciones maliciosas y mejora la seguridad del sistema operativo. También permite a los desarrolladores de aplicaciones no verificadas presentar sus aplicaciones a los usuarios de manera más segura.

## Consejo técnico

Si deseas instalar aplicaciones no verificadas en tu dispositivo Android, recuerda que debes esperar 24 horas después de habilitar las opciones de desarrollador y permitir paquetes no verificados. Luego, selecciona la opción de instalación temporal o indefinida y confirma que entiendes los riesgos.
