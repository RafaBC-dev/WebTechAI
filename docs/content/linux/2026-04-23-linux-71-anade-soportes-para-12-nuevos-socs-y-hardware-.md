# Linux 7.1 Añade Soportes para 12 Nuevos SoCs y Hardware ARM y RISC-V

**Fecha:** 2026-04-23
**Categoría:** linux
**Tags:** linux, microcontroladores, embebidos
**Título original:** Linux 7.1 Adds Support For 12 New SoCs, Other ARM & RISC-V Hardware

---

## Introducción

La versión 7.1 del kernel de Linux ha sido actualizada con soportes para 12 nuevos SoCs y hardware ARM y RISC-V. Esto significa que los desarrolladores pueden utilizar estos componentes en sus proyectos de manera más sencilla.

## ¿Qué es?

Linux es un sistema operativo de código abierto que se utiliza en una amplia variedad de dispositivos, desde servidores hasta dispositivos embebidos. El kernel de Linux es el núcleo del sistema operativo, y es responsable de gestionar los recursos del sistema y proporcionar una interfaz entre el hardware y el software.

## ¿Cómo funciona?

El kernel de Linux se compone de módulos que se cargan en memoria cuando se inicia el sistema. Cada módulo se encarga de una tarea específica, como gestionar la red o manejar la entrada/salida. Los desarrolladores pueden agregar nuevos módulos al kernel para agregar funcionalidades adicionales.

## ¿Por qué importa?

El soporte para 12 nuevos SoCs y hardware ARM y RISC-V en Linux 7.1 es importante porque permite a los desarrolladores utilizar estos componentes en sus proyectos de manera más sencilla. Esto puede acelerar el desarrollo de nuevos dispositivos y aplicaciones.

## Consejo técnico

Si estás desarrollando un proyecto que requiere el uso de hardware ARM o RISC-V, puedes utilizar el comando `make menuconfig` para configurar el kernel de Linux y agregar el soporte para el componente deseado.

```bash
make menuconfig
```
