# Corriendo Linux en un procesador de 1980: un logro impresionante

## Introducción

Los desarrolladores de Linux han estado eliminando soporte para procesadores antiguos, pero un ingeniero ha logrado correr el kernel más reciente en un Motorola 68008 de 1980. Esto demuestra la flexibilidad y la capacidad de adaptación de Linux.

## ¿Qué es?

El Motorola 68008 es un procesador de 8 bits diseñado en la década de 1980. A pesar de su antigüedad, un ingeniero ha logrado correr el kernel de Linux más reciente en él, utilizando un single-board computer llamado Mackerel-68k. Este proyecto demuestra la capacidad de Linux para funcionar en una amplia variedad de hardware.

## ¿Cómo funciona?

El Mackerel-68k es un single-board computer que utiliza un procesador Motorola 68008 overclockeado a 14 MHz y tiene 3,5 MB de RAM. A pesar de estas limitaciones, el kernel de Linux se puede cargar y funcionar en el sistema, proporcionando una shell minimalista de BusyBox. El proceso de carga del kernel es lento y el sistema se queda sin RAM rápidamente, pero es un logro impresionante.

## ¿Por qué importa?

Este logro es importante porque demuestra la capacidad de Linux para adaptarse a hardware antiguo y limitado. Esto puede ser útil para proyectos de retroinformática o para aquellos que buscan correr Linux en sistemas antiguos. Además, este proyecto puede inspirar a otros a explorar la capacidad de Linux para funcionar en hardware inusual.

## Consejo técnico

Si deseas correr Linux en un procesador antiguo, considera utilizar un single-board computer como el Mackerel-68k o el BeagleBone. Estos sistemas pueden ser una buena opción para proyectos de retroinformática o para aquellos que buscan correr Linux en hardware inusual.
