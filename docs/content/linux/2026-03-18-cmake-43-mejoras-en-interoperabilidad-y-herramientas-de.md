# CMake 4.3: Mejoras en interoperabilidad y herramientas de depuración

**Fecha:** 2026-03-18

## Introducción

La versión 4.3 de CMake ha sido lanzada, ofreciendo mejoras significativas en interoperabilidad y herramientas de depuración. Esto permitirá a los desarrolladores crear proyectos más complejos y manejables.

## ¿Qué es?

CMake es un herramienta de desarrollo de software y sistema de compilación que permite a los desarrolladores crear proyectos de manera eficiente y escalable. La versión 4.3 introduce el soporte para la especificación de paquetes comunes (CPS) para mejorar la interoperabilidad entre diferentes sistemas de compilación y administradores de paquetes.

## ¿Cómo funciona?

La especificación de paquetes comunes (CPS) es un formato JSON que describe los paquetes de software de manera clara y concisa. CMake 4.3 permite a los desarrolladores importar y exportar paquetes CPS, lo que facilita la creación de proyectos complejos y la interoperabilidad entre diferentes sistemas de compilación. Además, la herramienta de depuración de CMake ha sido mejorada para recopilar información de tiempo de ejecución y diagnóstico de sistemas.

## ¿Por qué importa?

Las mejoras en CMake 4.3 son importantes porque permitirán a los desarrolladores crear proyectos más complejos y manejables. La interoperabilidad mejorada entre diferentes sistemas de compilación y administradores de paquetes facilitará la creación de proyectos que involucran múltiples tecnologías y plataformas.

## Consejo técnico

Si estás trabajando con CMake, te recomendamos explorar la especificación de paquetes comunes (CPS) y cómo puedes utilizarla para mejorar la interoperabilidad en tus proyectos. Puedes encontrar más información sobre CPS en el repositorio de GitHub de Kitware.

```bash
cmake-instrumentation
```
