## Introducción

La reunión semanal de la ROS PMC (Revisión de Proyectos de ROS) ha tenido lugar el 2 de junio de 2026. En esta ocasión, se han discutido varias actualizaciones y tareas pendientes para el proyecto ROS 2. Entre ellas, se encuentra la necesidad de encontrar un revisor para las reglas de OpenEmbedded y la discusión sobre las reglas de Nix.

## ¿Qué es?

ROS 2 es una versión del framework de software de robótica ROS (Robot Operating System) que se enfoca en la creación de aplicaciones de software para robots y sistemas embebidos. Es una plataforma de código abierto que permite a los desarrolladores crear software de alta calidad y escalable para sistemas complejos.

## ¿Cómo funciona?

La ROS PMC es el grupo de revisión del proyecto ROS 2, que se encarga de garantizar la estabilidad y la calidad del código. El grupo se reúne semanalmente para discutir actualizaciones, tareas pendientes y problemas reportados. El proyecto ROS 2 utiliza herramientas como el buildfarm para automatizar la compilación y la depuración del código.

## ¿Por qué importa?

La actualización del proyecto ROS 2 es importante porque afecta a la comunidad de desarrolladores de software de robótica y sistemas embebidos. La discusión sobre las reglas de OpenEmbedded y Nix es crucial para garantizar la estabilidad y la seguridad del código. Además, la actualización del proyecto ROS 2 permitirá a los desarrolladores crear software más escalable y de alta calidad para sistemas complejos.

## Consejo técnico

Si estás trabajando con ROS 2, es importante revisar las reglas de OpenEmbedded y Nix para garantizar la estabilidad y la seguridad de tu código. Puedes consultar la documentación oficial de ROS 2 para obtener más información sobre cómo utilizar estas herramientas.

```bash
ros2 run rosidl_typesupport_fastrtps test_rosidl_buffer
```
