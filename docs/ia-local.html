IA Local
Corre IA en tu hardware

Sin APIs externas, sin coste por token, sin límites de privacidad. Esta guía cubre las herramientas principales para ejecutar LLMs localmente, qué modelos elegir según tu hardware y cómo integrarlos en tu flujo de trabajo.

1. Herramientas
Las cinco opciones principales para ejecutar LLMs localmente. Todas son gratuitas y funcionan en Windows, macOS y Linux.
Ollama
WindowsmacOSLinux
La opción más popular para desarrolladores
Herramienta de línea de comandos que gestiona la descarga, instalación y ejecución de modelos LLM. Expone una API REST compatible con OpenAI, lo que permite conectarlo con cualquier cliente sin cambios.
Instalación
Descarga desde ollama.com. En Linux:
curl -fsSL https://ollama.com/install.sh | sh
Descarga y ejecuta un modelo en un solo comando:
# Descarga y ejecuta Qwen3 9B ollama run qwen3:9b # Para hardware modesto ollama run llama3.2:3b
API local
Disponible en http://localhost:11434, compatible con OpenAI. Úsala desde Python, Aider, Continue u Open WebUI:
# Listar modelos ollama list # Ver modelos en ejecución ollama ps # Eliminar modelo ollama rm nombre-modelo
Ventajas
  • Catálogo enorme: Llama, Qwen, Mistral, Gemma, DeepSeek, Phi...
  • API compatible con OpenAI — funciona con Aider, Continue y Open WebUI sin configuración extra
  • Gestión automática de VRAM/RAM si el modelo no cabe en GPU
  • Mantiene el modelo en memoria entre llamadas
Limitaciones
Sin interfaz gráfica nativa. Para chatear visualmente necesitas Open WebUI u otro frontend.
ollama.com
LM Studio
WindowsmacOSLinux
Interfaz gráfica completa — la más amigable
Aplicación de escritorio para buscar, descargar y ejecutar modelos desde Hugging Face con una interfaz de chat integrada. Ideal para usuarios que prefieren no usar la terminal.
Instalación
Descarga el instalador desde lmstudio.ai y ejecuta. Sin línea de comandos. Busca modelos por nombre directamente desde la app.
Servidor local
Activa el servidor desde la pestaña "Local Server". Queda disponible en http://localhost:1234, compatible con OpenAI. Conéctalo a las mismas herramientas que usarías con Ollama.
Ventajas
  • Búsqueda integrada en Hugging Face con miles de modelos GGUF
  • Indicadores de VRAM en tiempo real al seleccionar modelo
  • Interfaz de chat sin configuración
Limitaciones
Gratuito solo para uso no comercial. Más pesado en recursos que Ollama.
lmstudio.ai
Jan
WindowsmacOSLinux
Alternativa open source a LM Studio — licencia MIT
Aplicación de escritorio 100% open source. Todo el código es auditable, los datos no salen de tu máquina y no hay restricciones de uso comercial.
Instalación
Descarga desde jan.ai. API local en http://localhost:1337, compatible con OpenAI.
Ventajas
  • 100% open source (MIT) — uso comercial libre
  • Sistema de extensiones
  • Sin telemetría — datos completamente locales
jan.ai
GPT4All
WindowsmacOSLinux
La mejor opción si no tienes GPU dedicada
Optimizado para correr en CPU. Incluye LocalDocs para chatear con tus propios documentos PDF y archivos de texto usando RAG, sin configuración adicional.
Instalación
Descarga desde gpt4all.io. Instalación como cualquier aplicación de escritorio.
LocalDocs — Chat con tus documentos
Ve a LocalDocs, crea una colección apuntando a tu carpeta. GPT4All indexa los archivos y usa RAG para responder preguntas sobre su contenido.
Caso de uso: documentación de un proyecto, manuales técnicos o código fuente — pregúntale directamente sin copiar y pegar.
gpt4all.io
Llamafile
WindowsmacOSLinux
Un archivo, cero dependencias, máxima portabilidad
Proyecto de Mozilla que empaqueta el modelo y el runtime completo en un único binario ejecutable. Sin instalar nada — copia el archivo a cualquier máquina y funciona.
Uso
Descarga el .llamafile desde GitHub Releases:
# Linux / macOS chmod +x mistral-7b.llamafile ./mistral-7b.llamafile # Windows: renombra a .exe y ejecuta
Al ejecutarlo abre automáticamente una interfaz web en localhost:8080.
Ventajas
  • Cero dependencias — todo en un solo archivo
  • Portabilidad total entre sistemas operativos
  • Ideal para entornos sin internet o air-gap
Limitaciones
Archivos muy grandes (incluyen el modelo). Menos modelos disponibles que en GGUF. Actualizaciones manuales.
GitHub — Mozilla-Ocho/llamafile
2. Modelos recomendados

3. Flujos de trabajo
Cómo combinar las herramientas locales con tu entorno de desarrollo.
Asistente de código en VSCode Continue + Ollama
Autocompletado y chat dentro del editor conectado a Ollama. Sin coste ni límites de peticiones.
1
Instala Ollama y descarga el modelo: ollama pull qwen3:9b
2
Instala la extensión Continue en VSCode desde el marketplace.
3
En la configuración de Continue añade proveedor Ollama, model qwen3:9b y base URL http://localhost:11434.
4
Usa Ctrl+I para inline chat y Ctrl+L para el panel lateral. Selecciona código y pregúntale directamente.
Agente de código con Aider Aider + Ollama
Aider edita archivos directamente desde terminal. Conectado a Ollama tienes un flujo de desarrollo completamente local.
1
Instala Aider: pip install aider-chat
2
Asegúrate de que Ollama está corriendo con el modelo.
3
Lanza con el modelo local:
aider --model ollama/qwen3:9b archivo.py
4
Para tareas de razonamiento sin prisa usa ollama/qwen3:27b. Para tareas rápidas el 9B es suficiente.
Tip: Añade /no_think al inicio del mensaje en Aider para desactivar el thinking mode en tareas simples.
Chat con documentos propios GPT4All LocalDocs
Consulta documentación técnica, manuales o código de un proyecto sin conexión a internet.
1
Instala GPT4All y descarga un modelo (Mistral 7B funciona bien para RAG).
2
Ve a LocalDocs y crea una colección apuntando a tu carpeta de documentos.
3
En el chat activa la colección y haz preguntas sobre el contenido indexado.
Interfaz web para Ollama Open WebUI + Ollama
Interfaz visual tipo ChatGPT para Ollama. La opción más completa para chatear con modelos locales.
1
Con Docker:
docker run -d -p 3000:8080 \ --add-host=host.docker.internal:host-gateway \ -v open-webui:/app/backend/data \ ghcr.io/open-webui/open-webui:main
2
Abre http://localhost:3000. Detecta Ollama automáticamente y muestra todos tus modelos.
3
Desde Open WebUI puedes gestionar conversaciones, subir archivos y comparar modelos en paralelo.
